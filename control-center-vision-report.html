שלב 1 — תיקוני באגים (מאמץ נמוך)
  • הוסף CSS עבור .ac-overlay ו-.ac-overlay-back ב-progress-map.html. ברירת מחדל display:none; JS הקיים מתניע.
  • הוסף JS לחלת ערכת נושא — ב-load קרא GET /user-profile, החל CSS variables על document.documentElement.
  • תקן devLabTest() — הוסף useLocal: window._useLocal || false לbody ב-progress-map.html:3027.
  • Build Prompt — הוסף כפתורי Apply/Copy לאחר קבלת תגובה מ-/build-prompt.
  • הגן על Progress Bar — Math.max(0, Math.min(100, (calls / (maxCalls||1)) * 100)).
שלב 2 — שיפורי UX (מאמץ בינוני)
  • Auto-refresh Alerts — polling כל 60 שניות ל-GET /control-center/events, או WebSocket subscription.
  • הוסף Health Score לWeb — ב-agents tab הצג healthScore (כבר בresponse) בצבע: ≥80 ירוק, 50-79 ענבר, <50 אדום.
  • שנה שם "ייצוא לקלוד" ל-"ייצוא prompt ל-Claude" + tooltip.
  • הוסף Analytics Range Filter לאפליקציה — Dropdown week/month/year בטאב אנליטיקה → GET /dashboard/analytics?range=.
שלב 3 — יישור ארכיטקטורה (מאמץ גבוה)
  • הפוך Web לפורטל — צמצם progress-map.html לדף נחיתה קצר: status card + כפתורי deep-link לאפליקציה לפי טאב.
  • הגדר deep-link scheme באפליקציה — Flutter handler ל-jarvis://control-center?tab= → navigate לtab index.
  • הסר כפילות תחזוקה — לאחר המעבר לפורטל, רוב progress-map.html ניתן לצמצום. תחזוקה = אפליקציה בלבד.
  • חזק "בחינת יכולות" באפליקציה — Feature Lab מסודר, שיחת בדיקה בולטת, health score עם drill-down.
