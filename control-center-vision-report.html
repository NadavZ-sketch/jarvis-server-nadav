שני "מרכזי שליטה" קיימים היום בנפרד — אחד באפליקציה (Flutter) ואחד ב-Web. כיום הם כפילים זה של זה. החזון מגדיר תפקידים משלימים: האפליקציה היא המקום לראות ולבחון את יכולות ג'רביס; ה-Web הוא שער/הפניה לאזור הוויזואלי של מרכז השליטה.

📱 אפליקציה

מרכז בחינת היכולות — לראות ולבחון את ג'רביס
  • רשימת 24 סוכנים עם מדדים חיים + health score
  • כרטיס סוכן: שיחה ישירה (chat), Prompt Builder, חיבורים
  • Feature Lab / E2E / סריקה — בחינה בפועל
  • Command bar בשפה טבעית

🌐 Web

שער / הפניה — מצביע לאמצעים הוויזואליים
  • נקודת כניסה מהירה דרך דפדפן (/progress-map)
  • מפנה אל / מטמיע את האזור הוויזואלי של האפליקציה
  • מקור אמת אחד — בלי תחזוקה כפולה
  • היום מושק חיצונית מההגדרות באפליקציה (_openProgressMap)

⚠️ מצב נוכחי

ה-Web (progress-map.html, 3103 שורות) וה-Flutter (progress_map_screen.dart) מממשים את אותו דשבורד פעמיים — אותם 6 טאבים, אותם endpoints. כל שינוי צריך תחזוקה בשני מקומות. חלק מהפיצ'רים מפותחים בצד אחד ולא בשני (פערי parity).

✅ מצב יעד

האפליקציה היא הבית של בחינת היכולות (UX מובייל מלא). ה-Web הופך לשער דק שמפנה/מטמיע את האזור הוויזואלי — מקור אמת אחד, פחות באגים, פחות תחזוקה.

CSS חסר — .ac-overlay שבור במובייל
לחץ לפרטים ▾
ה-HTML מגדיר div.ac-overlay ו-button.ac-overlay-back (:1033-1035) לפאנל מובייל. ה-JS מוסיף class .open (:2567,:2572) — אבל אין CSS ל-selectors האלה. אין position/z-index/display → תצוגת פרטי הסוכן במובייל בלתי-נראית.

תיקון: הוסף .ac-overlay{position:fixed;inset:0;z-index:1000;...} + .ac-overlay.open{display:block}.

לחץ לפרטים ▾

תיקון: data-theme על <html> + CSS variable overlays per theme, מיושם ב-init מ-localStorage/profile.
